patch_quality 标准：close 基本接近 gold patch；partial 改到相关区域但不完整；poor 明显跑偏；empty 没有有效 patch；env_only 主要是环境失败。

          
SFT 可用标准：通常只有 resolved 或人工确认高质量正样本才设为 true；失败 patch 默认 false。

          
DPO 可用标准：有 gold patch、agent patch 非空、不是环境错误，并且 agent 未解决时，一般可设为 true。

          
环境错误规则：如果是 Docker、dataset、dependency 等环境问题，选择对应 EnvironmentError，patch_quality 用 env_only，SFT/DPO 都设为 false。

          
close 但 failed：如果 patch 很接近 gold patch 但 run 未 resolved，SFT 保持 false，DPO 通常 true，notes 写清楚差异。
